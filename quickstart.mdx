---
title: "Get Sketchbook Map running locally in five minutes"
description: "Learn how to clone the repo, configure your Mapbox token in config.js, start the Node server, and open your sketch collection on the map."
---

Sketchbook Map is a Mapbox GL JS application that plots hand-drawn sketchbook pages and type specimen photos at their real-world locations. Running it locally takes about five minutes: you clone the repo, drop in your Mapbox token, and start the Node server. Once it's running, you can browse your collection on an interactive map, use the admin panel to place new sketches, and toggle between the two collections from the bottom of the screen.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** — runs the local development server (`server.js`)
- **Python 3** — required for the image processing scripts in `scripts/`
- **cwebp** — converts images to WebP thumbnails used by the map

## Set up the project

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine and move into the project directory.

    <CodeGroup>
      ```bash Terminal
      git clone https://github.com/notmattgrossman/drawing-map.git
      cd drawing-map
      ```
    </CodeGroup>
  </Step>

  <Step title="Create your config file">
    The app reads your Mapbox token from `config.js`, which is not committed to the repository. Copy the provided example file to create your own:

    <CodeGroup>
      ```bash Terminal
      cp config.example.js config.js
      ```
    </CodeGroup>

    The example file looks like this:

    ```javascript config.example.js
    // Configuration file for API keys and sensitive data
    // This file is ignored by git - never commit it!
    // On GitHub Pages, this is generated from environment variables

    const CONFIG = {
        MAPBOX_ACCESS_TOKEN: ''
    };

    // Make config available globally
    if (typeof window !== 'undefined') {
        window.CONFIG = CONFIG;
    }
    ```
  </Step>

  <Step title="Add your Mapbox token">
    Open `config.js` and paste your Mapbox public access token as the value for `MAPBOX_ACCESS_TOKEN`:

    ```javascript config.js
    const CONFIG = {
        MAPBOX_ACCESS_TOKEN: 'pk.your_token_here'
    };
    ```

    <Note>
      Mapbox access tokens are free for personal use. You can create one at [mapbox.com](https://account.mapbox.com/access-tokens/) by signing up for a free account. Use a **public** token (prefixed with `pk.`).
    </Note>
  </Step>

  <Step title="Start the server">
    Sketchbook Map has no npm dependencies — the server uses only Node.js built-in modules. Start it directly:

    <CodeGroup>
      ```bash Terminal
      npm start
      ```
    </CodeGroup>

    You should see output confirming the server is running:

    ```
    ╔════════════════════════════════════════════╗
    ║   Sketchbook Map Server Running            ║
    ╚════════════════════════════════════════════╝

      Admin Panel:  http://localhost:3000/admin/
      Public Map:   http://localhost:3000/
    ```
  </Step>

  <Step title="Open the app in your browser">
    With the server running, open your browser and navigate to:

    - **Public map** — `http://localhost:3000`: the interactive map showing your sketch and type specimen collections
    - **Admin panel** — `http://localhost:3000/admin`: a drag-and-drop tool for placing new sketches on the map

    Use the toggle at the bottom of the screen to switch between the **Sketchbooks** and **Type Specimens** collections.
  </Step>
</Steps>
